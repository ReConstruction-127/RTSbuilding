RTS / GUI / Craft Terminal / JEI behavior notes
Updated: 2026-03-10

1. RTS enabled, normal container GUI open

- Shift + click on a container slot item:
  Import the whole clicked stack into RTS linked storage.
  This bypasses vanilla quick-move.

- Click on a container slot item:
  Keep vanilla click behavior.

- Shift + click on a player inventory or hotbar slot item:
  If the current screen is the player inventory opened by E, or the vanilla crafting table screen,
  import the whole clicked stack into RTS linked storage.
  On other container screens, do not intercept it.
  Let vanilla quick-move handle the transfer inside the menu.

- Click on a player inventory or hotbar slot item:
  Keep vanilla click behavior.

- If the open GUI is a CraftingMenu and the clicked slot is the crafting result slot:
  Normal click first behaves like vanilla result pickup, then sends an RTS refill.
  The refill now pulls ingredients from:
  linked storage first, then player main inventory.
  The refill does not use the hotbar.

- Shift + left click on the crafting result slot:
  Import crafted output directly into linked storage and keep refilling/crafting in the same action.
  Refill source is:
  linked storage first, then player main inventory.
  Hotbar is not used.

- Shift + right click on the crafting result slot:
  Currently intercepted but does not perform import.

2. RTS Craft Terminal

- Click on the crafting result slot:
  First behaves like vanilla result pickup, then the server immediately refills the grid using the saved blueprint.
  Refill source is:
  linked storage first, then player main inventory.
  Hotbar is not used.

- Shift + left click on the crafting result slot:
  Batch-import crafted output into linked storage and keep refilling/crafting while the recipe can continue.
  Refill source is:
  linked storage first, then player main inventory.
  Hotbar is not used.

- Shift + right click on the crafting result slot:
  Currently intercepted but does not perform import.

- Click on crafting grid or player inventory slots:
  Keep vanilla click behavior.

- Shift + click on crafting grid or player inventory slots:
  Import the clicked stack into linked storage instead of using vanilla quick-move.

3. Linked panel in RTS Craft Terminal

- Left click a linked item entry:
  Pick up as many as possible into carried stack, up to max stack size.

- Right click a linked item entry:
  Pick up one item into carried stack.

- Left click linked panel with a carried stack:
  Return the whole carried stack to linked storage.

- Right click linked panel with a carried stack:
  Return one item from the carried stack to linked storage.

4. Overlay linked storage bar on normal container screens

- Left click an RTS overlay storage entry:
  Pick up as many as possible into carried stack.

- Right click an RTS overlay storage entry:
  Pick up one item into carried stack.

- Shift + click an RTS overlay storage entry on non-E/non-crafting-table container screens:
  Try a vanilla-like container transfer into the currently opened menu first.
  If anything cannot fit into the container side, the remainder falls back to player inventory.

- Shift + click an RTS overlay storage entry on the player inventory screen or crafting table screen:
  Insert the extracted stack into player inventory instead of picking it to carried.

- Left click return slot or overlay grid while carrying:
  Return the whole carried stack to linked storage.

- Right click return slot or overlay grid while carrying:
  Return one item from the carried stack to linked storage.

5. JEI plus button

- Click JEI +:
  Clear the current crafting grid first, then transfer one recipe set into the grid.

- Shift + click JEI +:
  Clear the current crafting grid first, then repeatedly fill the grid as much as possible.

- JEI transfer source:
  linked storage first, then player main inventory.
  Hotbar is not used.

- Old grid contents removed by JEI transfer:
  Try to return to linked storage first.
  If linked storage cannot fully accept them, the remainder goes back to player inventory.

6. Notes about counting and source boundaries

- The storage page can include player main inventory counts for browsing/crafting availability.
- The actual refill/extract logic does not blindly trust displayed counts.
- Refill/extract uses real extraction in source order:
  linked storage first, then player main inventory.
- Player hotbar is intentionally excluded from refill.
- This avoids the common double-count problem where displayed RTS totals may include player inventory as part of the network view.

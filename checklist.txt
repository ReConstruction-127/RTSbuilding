RTS Building - Sims4 Camera Checklist
Date: 2026-03-01

Completed (Sims4-Only + Local Mirror Camera)
- Switched camera attachment to local mirror camera entity (client render-only).
- No longer attaches view directly to network-synced server camera entity.
- Server camera remains authoritative for validation/range and move handling.
- Client now:
  - receives server snapshot state
  - runs local prediction + EMA smoothing
  - drives local mirror camera every tick and render frame
- Kept fixed packet cadence (one C2S move packet per client tick while RTS enabled).
- Kept rotate capture mode and RTS input/hud suppression.
- Added rotation sensitivity adjustment:
  - [ decrease sensitivity
  - ] increase sensitivity
- Left-drag pan is enabled again.
- A/D direction fixed: A moves left, D moves right.
- Left/right drag direction kept as current "positive" mapping (not inverted).
- Fixed RTS cursor raycast horizontal inversion:
  - root cause: X NDC was applied to a yaw-derived left-vector basis
  - fix: invert X contribution (`-nx`) in cursor ray direction calculation
  - applied in both `BuilderScreen` and `RtsBoundaryRenderer` to keep pick/highlight consistent
- Reworked RTS UI layout for clearer workflow:
  - top bar now has exactly 3 mode buttons: `BREAK`, `LINK`, `DRAG`
  - bottom panel now follows: sort buttons -> category column -> search + paged item storage grid
- Implemented server-backed storage category filter (namespace-based):
  - request payload now carries `category`
  - server applies category + search + sort + page in one pipeline
  - response now includes selected category and available category list
- Implemented in-UI category interaction:
  - category list can be selected by click
  - category list supports scroll buttons and mouse wheel scroll
- Implemented item selection to start placing flow:
  - clicking an item in storage grid marks it selected
  - selection switches mode to `BUILD` (place intent state on client side)
- Link storage path validated with new UI:
  - enter `LINK`, click world storage block, storage panel refreshes from server
- Implemented RTS place flow (selected block placement):
  - clicking item in storage grid selects place target and switches to `BUILD`
  - in `BUILD`, left-click world block to place selected block at target location
  - server validates mode/link/item, consumes 1 item from linked storage, places block, refreshes storage page
- Input behavior refinement while searching:
  - removed `Q/E` quarter-rotation shortcut
  - when search box is focused, keyboard WASD/arrow movement is suppressed
- Bottom panel compactness + readability update:
  - reduced panel height to show more world view
  - enlarged grid cell pitch
  - item count text moved to top-left overlay with shadow for visibility
- Further compacted bottom UI to roughly half-height again:
  - reduced bottom panel from 150 -> 76
  - re-packed sort controls into a denser vertical block
  - kept category/search/pager/grid while increasing slot cell size to 22 pitch
- Fixed item count readability layering:
  - count now renders with dark backing after item render, avoiding icon-over-text overlap
- Added top `ROTATE` mode:
  - supports clockwise/counter-clockwise rotation steps for placement orientation
  - placement request now carries hit vector + rotation steps
- Upgraded place pipeline to full use-on flow (`useItemOn` style context):
  - respects block interaction/placement behavior beyond default block state
  - supports bucket-like item behavior (e.g., water bucket placing fluid)
  - applies post-place quarter rotation to rotatable block states
- Implemented RTS break execution:
  - in `BREAK` mode, only blocks tracked as player-placed are insta-broken
  - broken block refund is inserted back into linked storage (overflow drops to player)
  - storage page refreshes after break/place changes
- Updated break operation to hover + `K`:
  - removed mouse-click break trigger
  - in `BREAK` mode, pressing `K` on hovered block executes break request
- Added adjustable bottom panel height:
  - bottom panel now supports runtime height adjustment via compact `+/-` controls in sort area
  - supports short view for observation and extended view for search/list browsing
- Restored RTS right-click item interaction flow:
  - when an item is selected, world right-click uses that item on the cursor raycast block
  - selected item is rendered next to mouse cursor as a live placement/use cursor
  - camera rotate-capture moved to middle mouse drag to preserve right-click item use
- Simplified top mode bar to only `LINK` and `ROTATE` (removed other top buttons).
- Updated drag input mapping:
  - removed left-button drag behavior
  - switched drag-pan to middle-button drag
- Updated break trigger to keyboard:
  - break is now `hover target + K` (no mouse-click break)
  - break/place server side no longer depends on old mode toggles so the new 2-mode UI remains functional
- Search interaction refinement:
  - middle-button drag pan remains available even while search box is focused
  - added search-cancel key: `Esc` clears search text and exits search focus without closing RTS screen
- Container-screen linked inventory overlay:
  - when RTS is enabled and a vanilla container screen is open, a compact RTS linked bar is rendered at screen bottom
  - overlay supports page switching (`<` `>` and mouse wheel inside panel)
  - overlay supports clicking linked entries to select RTS item while container remains open
- Container drag support from linked overlay:
  - clicking linked overlay entries now requests server transfer into the container carried stack
  - left-click picks up stack-sized amount, right-click picks up 1
  - carried stack can then be dragged into crafting/container slots like normal inventory items
- Unified action model to `INTERACT` (removed `BUILD` mode path):
  - top bar now uses `INTERACT` / `LINK` / `ROTATE`
  - selected-item flow no longer switches to old `BUILD`; it stays in `INTERACT`
- Updated world mouse bindings:
  - left click: break placed block on hover ray target
  - right click: interact/use selected item on hover ray target (fallback empty-hand interact)
  - middle click: clear selected RTS item and force empty-hand `INTERACT` state
- Fixed INTERACT right-click order for utility blocks:
  - server now attempts empty-hand block interaction first
  - if not consumed, then falls back to selected linked-item use-on
  - resolves furnace/crafting-table style GUI interactions being blocked by selected item use
- Reduced accidental item drops on container close:
  - added linked-overlay carried-return packet path
  - if carried stack came from RTS linked overlay and container is closing, stack is returned to linked storage instead of dropping
- Fixed container GUI flicker-close in RTS interact flow:
  - after successful interact/open, container reachability checks are disabled on the opened menu
  - prevents furnace/crafting-table style screens from opening for one frame then snapping back to RTS builder screen
- Improved linked overlay pickup responsiveness:
  - clicking linked overlay entries now applies an immediate local carried-stack preview
  - removes the "must hold mouse to see item follow cursor" feel caused by waiting for server pickup sync
- Limited RTS interaction-key cancellation scope:
  - attack/use key mapping suppression now only applies in world/BuilderScreen
  - container GUI input path is no longer globally suppressed while RTS is enabled
- Regression fix (container flicker-close and chest reopen loop):
  - reverted interaction-key suppression back to RTS-global cancellation to prevent immediate second world-use after GUI opens
  - interact pipeline now detects actual menu-open by comparing `player.containerMenu` before/after use
  - when a menu has opened, selected-item fallback use is skipped (prevents open->second-use->close race)
- Validation relaxation fix for 1.21 menus:
  - replaced ineffective `AbstractContainerMenu.checkReachable` reflection path
  - now relaxes opened menu validation by patching menu fields (`ContainerLevelAccess -> NULL`, `Container -> always-valid delegate`) where available
- Upgraded container RTS linked overlay closer to vanilla slot flow:
  - expanded linked grid from 2 rows to 4 rows (9x4 = 36 entries per page, matching storage page size)
  - added explicit mouse press/release capture for overlay interactions to remove "must hold mouse" behavior caused by release-side container handling
  - pickup requests now respect carried-stack compatibility/capacity before sending, mirroring normal slot merge constraints
- Added 5-slot "Return to Storage" import lane:
  - new return lane accepts carried items from either player inventory or opened container inventory
  - dropping carried stack into return lane sends import request back to linked storage (`C2SRtsReturnCarriedPayload`) and clears/shrinks carried stack locally
  - return lane keeps short-lived queued previews for recent imports to support faster batch workflow
- Multi-link storage support:
  - `LINK` mode now toggles the clicked storage block in a linked-storage set instead of replacing a single target
  - storage page aggregation now merges item counts across all currently linked storages
  - place/pickup/break-refund/return-import paths now operate against the linked storage pool (not single storage)
- Linked-storage persistence across RTS toggles:
  - exiting RTS no longer clears linked storage session state
  - re-entering RTS restores linked set and immediately re-requests page data using saved session filters/page/sort
- Blue linked-storage highlight:
  - server now syncs linked storage block positions to client in storage page payload
  - RTS world renderer draws blue line-box highlight around all linked storage blocks
- Right-click interaction split (click place vs hold-drag rotate):
  - right mouse press now starts a pending world action instead of immediate place/use
  - right mouse drag in world feeds camera rotation (`queueRotateDrag`)
  - right mouse release only triggers place/use when no drag-rotation happened (click-only place)
- Added top-bar sensitivity button for rotation + zoom:
  - new `SENS x..` button cycles input sensitivity presets
  - sensitivity scale now applies to both drag-rotate input and mouse-wheel zoom/dolly input
  - existing `[` `]` hotkeys for base rotate sensitivity remain available
- Container-break absorb behavior:
  - breaking tracked container blocks now extracts internal contents before removal
  - extracted contents are inserted into linked storage pool to avoid dropping items on the ground
- Overflow fallback + hinting for absorb/import:
  - when linked storage is full during absorb/import, overflow first attempts player inventory insert
  - if player inventory is also full, remainder is dropped on ground
  - action-bar hint is shown for overflow cases, including moved-to-inventory and dropped counts
- Added server-side placed-block tracker foundation:
  - `PlacedBlockTrackerData` SavedData (dimension-level persistence)
  - marks on player place (`EntityPlaceEvent`, `EntityMultiPlaceEvent`)
  - clears on player break (`BreakEvent`)
- Added mining loadout foundation:
  - roles: Pick/Shovel/Axe/Hoe
  - per-player slot references in persistent data (`MiningLoadoutState`)
  - matching check via stored slot fingerprint
- Added RTS break policy foundation:
  - player-placed blocks can be insta-break
  - non-player-placed blocks require loadout/tool checks (`RtsBreakPolicy`, `RtsMiningRules`)
- Compile verification passed: .\gradlew.bat compileJava

Completed (Fast Safety Fixes, 2026-03-02)
- Added server-side RTS action guards:
  - world-target actions now require active RTS session + chunk loaded + mayInteract + in-camera action radius
  - applied to `linkStorage`, `placeSelected`, and `breakPlaced`
- Added RTS-active guard for linked-storage transfer paths:
  - `pickupLinkedToCarried` and `returnCarriedToLinked` now require active RTS session
- Added cross-dimension linked-storage sanitization:
  - linked storage set is auto-cleared when player dimension differs from stored linked dimension
  - prevents stale cross-dimension same-coordinate relinking/accidental access
- Added linked-dimension session tracking:
  - session now records the dimension key when linking storage
  - clears tracked dimension when last link is removed
- Added logout cleanup:
  - `RtsStorageManager` session is removed on player logout to avoid stale session accumulation
- Added `RtsCameraManager` helpers for server validation:
  - `isActive(ServerPlayer)`
  - `isWithinActionRadius(ServerPlayer, BlockPos)`
- Fixed linked-container break absorb item loss:
  - break absorb now excludes the container being broken from linked insert targets
  - recovered items now route to other linked storages first; if no capacity, fallback path applies
  - breaking a linked storage block now unlinks that block immediately to avoid stale link entries
- Fixed persistent RTS item-count layering issue:
  - item-count text now renders in a forced foreground Z layer in `BuilderScreen` storage grid
  - container RTS linked overlay (`RtsClientInputGate`) uses the same foreground count overlay rendering
  - resolves icon-over-count overlap so remaining quantities stay readable
- Improved bucket/use-on interaction in RTS:
  - right-click world raycast now includes fluid source targeting when selected RTS item is a bucket
  - enables expected bucket source targeting for pickup/interaction
  - server-side item use pipeline now falls back from `useItemOn` to `useItem` for bucket items when needed
  - ensures water/lava buckets can both place and pick up fluids through RTS interaction flow
- Added hold-to-mine flow with selectable hotbar tool:
  - new C2S mine payload (`C2SRtsMinePayload`) with start/abort action and selected tool slot
  - in RTS world area, `LMB press` starts mining and `LMB release` aborts mining (vanilla-like hold behavior)
  - server mining uses `ServerPlayerGameMode.handleBlockBreakAction` for normal block hardness/progress
  - player-placed tracked blocks still use original instant-break absorb path
- Added hotbar row above RTS storage panel:
  - shows current player hotkeys (1-9) with live item icons/counts
  - click a hotbar slot or press number key to choose current mining tool slot
  - world interaction area now excludes this hotbar row to avoid click conflicts
- Mining flow fix + UI refinement:
  - fixed non-working hold-mine by adding continuous mining packets while LMB is held (`BuilderScreen.tick -> continueMining`)
  - server mining path now supports continuous START-destroy updates instead of single-shot behavior
  - moved tool hotbar into bottom storage panel (under search row) for visual consistency
  - added 5 RTS quick slots next to tool row for frequently used RTS items
  - quick slot behavior: Shift+LMB on quick slot = assign current selected RTS item; LMB = select quick item
- Added auto-store toggle for mined drops:
  - new top-bar toggle button: `AUTO STORE: ON/OFF`
  - state synced C2S via `C2SRtsSetAutoStorePayload`
  - storage page payload now includes `autoStoreMinedDrops` state for client sync
  - when enabled, mined block drops near target are auto-inserted into linked storage as capacity allows
- Fixed RTS mining/interaction regressions:
  - hold-mine protocol now uses START on press and STOP on release (with monotonic mining sequence id)
  - removed per-tick repeated START reset behavior that prevented normal block break progress
  - right-click now supports placing/using current player hotbar item through the RTS cursor-hit position
  - cursor preview now falls back to current main-hand item when no linked-storage item is selected
- Interaction target/range behavior update:
  - removed RTS anchor-radius limit from world-target actions (mine/place/interact/link/break)
  - world actions now rely on RTS cursor ray hit + chunk loaded + permission checks
  - removed bucket `useItem` fallback that caused interactions to happen in front of player body instead of ray hit

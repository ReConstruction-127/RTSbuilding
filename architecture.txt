3. Layered Responsibilities
==================================================
Bootstrap layer:
- RtsbuildingMod registers entity, example content, and game events.
- On logout/dimension-change it stops RTS camera.
- On server tick it calls RtsStorageManager.tickMining.

Network layer:
- RtsPayloadRegistrar registers all play payloads.
- RtsNetworkHandlers routes C2S actions to server managers and S2C to client controller.

Client runtime layer:
- ClientRtsController stores all RTS client-side state.
- BuilderScreen owns RTS UI layout/input behavior.
- RtsClientInputGate suppresses vanilla interaction/hud/hand while RTS is active.
- RtsBoundaryRenderer draws action bounds, linked storage outlines, hovered target outline.

Server runtime layer:
- RtsCameraManager controls RTS camera lifecycle and movement constraints.
- RtsStorageManager is authoritative for link/place/interact/mine/store/refund logic.

Persistence layer:
- PlacedBlockTrackerData stores world placed block positions.
- MiningLoadoutState stores per-player assigned mining tool slots (persistent NBT).

4. RTS Camera Lifecycle
==================================================
Input trigger:
- Key R in ClientKeyMappings.
- ClientInputHandler sends C2SRtsToggleCameraPayload on key release with cooldown.

Server start:
- RtsCameraManager.start:
  - uses player position as anchor.
  - spawns RtsCameraEntity near anchor.
  - records session (camera UUID, anchor, yaw/pitch/height).
  - calls RtsStorageManager.onRtsEnabled.
  - sends S2CRtsCameraStatePayload enabled=true.

Server stop:
- RtsCameraManager.stop:
  - discards camera entity.
  - removes session.
  - sends S2CRtsCameraStatePayload enabled=false.
  - calls RtsStorageManager.onRtsDisabled.

Client activation:
- ClientRtsController.applyServerCameraState(enabled=true):
  - stores previous camera and visual options.
  - forces first person + disables bob and fov effect scale.
  - opens BuilderScreen.
  - initializes local predicted camera state and storage UI state.
  - creates local mirror camera and requests first storage page.

Client deactivation:
- restores previous camera/options, closes BuilderScreen, clears RTS state.

6. Current Mouse Interaction Logic
==================================================
Left click:
- In LINK_STORAGE mode and world area: raycast block and link/unlink storage endpoint.
- In other modes and world area: start remote mining.
- Hold left mouse keeps mining active; release aborts.

Right click normal (no Alt):
- If mode is LINK_STORAGE: ignore right-click.
- If RTS selected item exists:
  - entity target -> interactEntityWithPinnedItem.
  - block target -> placeSelected(hit, forcePlace=Shift).
- If no RTS selected item:
  - entity target and main-hand non-empty -> interactEntityWithToolSlot.
  - block target and main-hand non-empty -> placeSelected(hit, forcePlace=Shift).
  - block target and main-hand empty -> interactEmpty(hit).

Alt + right click:
- Opens radial interaction wheel (if valid target and valid options exist).
- Wheel options include:
  - hotbar tool slots with non-empty stacks,
  - non-empty RTS pin items.
- Wheel selection sends C2SRtsInteractPayload (tool source or pin source).

Raycast:
- Uses custom cursor ray from camera with max distance 128.
- Chooses entity target when entity hit is closer/equal to block hit.
- For bucket-like contexts, fluid SOURCE_ONLY path is enabled on right-click target pick.

8. Server Core (RtsStorageManager)
==================================================
Session model per player:
- mode
- linked storage positions/names/dimension
- page/search/category/sort/ascending
- autoStoreMinedDrops
- current mining target/progress/tool slot

8.1 Linking storage
- Validates target via canAccessWorldTarget.
- Finds IItemHandler capability at block.
- Toggles position in linked list.
- Enforces same-dimension by linkedDimension + sanitizeSessionDimension.
- Refreshes storage page.

8.2 Storage page aggregation
- If RTS not active or no linked storages -> empty page payload.
- Resolves active linked handlers.
- Aggregates item counts across handlers.
- Builds categories (all + namespaces).
- Applies search + category filter.
- Applies sort (quantity/mod/name).
- Paginates with PAGE_SIZE=90.
- Sends S2CRtsStoragePagePayload.

8.3 placeSelected (normal right-click server side)
Branch A: itemId is blank (use player's current main-hand)
- useItemOn with optional temporary shift (forcePlace).
- if not consumed -> fallback useItem under temporary interaction position.
- if menu opened, container validation is relaxed.

Branch B: itemId is not blank (use linked storage material)
- extracts one matching item from linked handlers.
- temporarily injects extracted stack into main hand.
- useItemOn first (respect forcePlace).
- if not consumed -> fallback useItem under temporary interaction position.
- refunds remainder stack to linked storage.
- marks placed position and applies rotation steps when placement happened.
- refreshes page.

8.4 interactTarget (Alt wheel server side)
Source types:
- SOURCE_TOOL_SLOT
- SOURCE_PIN_ITEM

Target types:
- Entity by entityId
- Block by clickedPos + hit data

Tool-slot block interaction order:
1) useItemOn shift=false
2) useItem shift=false
3) useItemOn shift=true
4) useItem shift=true

Pin-item block interaction order:
1) useItemOn secondary=false
2) useItem secondary=false
3) useItemOn secondary=true
4) useItem secondary=true

Both paths use temporary interaction position around target.
After interaction, storage page refreshes.

8.5 Cursor carry transfer APIs
- pickupLinkedToCarried(itemId, amount): extract from linked and merge into carried cursor stack.
- returnCarriedToLinked(itemId, amount): split from carried and store back to linked.
- Overflow fallback path: linked -> inventory -> world drop.

8.6 Mining
Entry points:
- mine(... start=true/false ...)
- tickMining called every server tick from mod event.

Flow:
- beginRemoteMining sets mining state.
- tickActiveMining computes destroy step and updates break stage.
- stage updates sent via S2CRtsMineProgressPayload.
- when progress >= 1.0, destroy block.
- if autoStoreMinedDrops true, absorb nearby ItemEntity drops into linked storage.

Important mining behavior:
- computeRemoteDestroyStep temporarily sets onGround=true.
- if player container carried stack is not empty, that carried stack is temporarily placed into selected tool slot for speed/harvest calculation and then restored.

13. Known Risk Areas and Cleanup Candidates
==================================================
- Unused code paths/methods currently exist:
  - BuilderScreen.tryDirectToolInteraction
  - RtsStorageManager.refundItem
  - ClientRtsController.breakPlaced path not used by current BuilderScreen mining flow
- Server stores mode but does not hard-gate all world mutations by mode.
- Reflection-based menu validation relaxation is powerful but fragile on upstream changes.

2026-03-04 - Crash fix: Render buffer state bug in shape ghost overlay (IllegalStateException: Not building!)

Issue
- Client crashed on render thread with:
  - `java.lang.IllegalStateException: Not building!`
  - stack points to `LevelRenderer.renderLineBox(...)`
  - crash site: `RtsBoundaryRenderer.onRenderLevel(...)`.

Root cause
- In `onRenderLevel`, code requested two VertexConsumers up front:
  - `RenderType.lines()`
  - `RenderType.debugFilledBox()`
- Under BufferSource state switching, obtaining a second render type can end/switch active builder.
- The previously cached `lineBuffer` then became stale/non-building but was still used for line writes,
  producing the `Not building!` exception.

Fix strategy
- Stop holding mixed render-type consumers across state switches.
- Draw per render-type in grouped phases and reacquire consumer after switching type.

Code changes
1) `RtsBoundaryRenderer.onRenderLevel(...)`
- Removed eager `fillBuffer` acquisition.
- Kept initial line rendering with `lineBuffer` only.
- Changed ghost render call to pass `bufferSource` instead of pre-cached consumers.
- End-batch order set explicitly after rendering:
  - `endBatch(RenderType.lines())`
  - `endBatch(RenderType.debugFilledBox())`

2) `renderShapeGhostPreview(...)`
- Signature changed:
  - from `(Minecraft, PoseStack, VertexConsumer lineBuffer, VertexConsumer fillBuffer)`
  - to `(Minecraft, PoseStack, BufferSource bufferSource)`
- Rendering split into two passes:
  - pass A: acquire `debugFilledBox` consumer and render all translucent fills
  - pass B: reacquire `lines` consumer and render all outlines
- Prevents stale-consumer usage across RenderType changes.

Compatibility impact
- No protocol/server changes.
- Pure client render-path correction.
- Preserves existing ghost visual behavior (fill + outline + ready-confirm color scheme).

- During non-90-degree shape rotation, generated structures (especially SQUARE/BOX) showed visible holes/gaps.
- Root cause: previous generation used forward mapping (`rotatePlaneOffset` + integer rounding) from source grid to destination grid.

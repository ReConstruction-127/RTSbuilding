RTSbuilding Optimization Task Breakdown (2026-03-06)

Context
- User requests a coordinated set of RTS/storage/crafting/JEI/shape-build behavior fixes.
- This document structures requirements into actionable items with acceptance checks.

Item 1 - Hotbar source must participate in chain build
Symptom
- Selecting a block from hotbar cannot use chain/shape placement flow.
Root Cause (expected)
- Shape flow currently requires RTS selected item, not tool-slot source.
Change Scope
- BuilderScreen shape start/confirm path should accept hotbar tool-slot block as source.
- Preview and confirm conditions should support either RTS selected item OR hotbar block source.
Acceptance
- With no RTS selected item, but hotbar block selected, shape preview appears and confirm places shape.

Item 2 - Q quick drop should work for hotbar even without linked storage
Symptom
- Without linked storage, Q cannot drop hotbar item in RTS.
Root Cause (expected)
- Quick-drop server path hard-requires linked storages and extraction excludes hotbar.
Change Scope
- Quick-drop should allow inventory/hotbar source fallback when no linked storage.
Acceptance
- No linked storage: pressing Q with hotbar item drops 1 item near cursor target.

Item 3 - Funnel should work to player inventory without linked storage
Symptom
- Funnel import cannot move items to player inventory when no linked storage.
Root Cause (expected)
- Funnel tick exits early if no linked positions.
Change Scope
- Funnel processing should still run with empty linked handlers.
- Destination priority: linked storages first, then player backpack, then buffer.
Acceptance
- No linked storage + funnel target with drops: items move into player inventory.
- With linked storage: items prefer linked storage first, inventory as fallback.

Item 4 - JEI '+' transfer must work in both RTS and normal modes
Symptom
- JEI '+' currently only usable in RTS storage-linked flow; blocks normal crafting behavior.
Root Cause (expected)
- Custom transfer handler registered too broadly for all CraftingMenu and returns user error.
Change Scope
- Restrict custom handler to RTS craft terminal menu path only.
- Do not block vanilla handler in normal CraftingMenu contexts.
Acceptance
- Normal crafting screen: JEI '+' works as vanilla.
- RTS craft terminal: JEI '+' still works with RTS backend.

Item 5 - RTS craft terminal output rendering issue (result appears invisible)
Symptom
- Crafting result is present/usable but not visible in RTS craft terminal output slot.
Change Scope
- Ensure terminal menu validation remains relaxed.
- Add defensive result-slot rendering fallback in RTS terminal screen.
Acceptance
- In RTS craft terminal, crafting result is visibly rendered in output slot.

Item 6 - Terminal storage page should not include player inventory entries
Symptom
- Terminal linked-storage panel shows items from player inventory; causes confusion.
Root Cause (expected)
- requestPage always aggregates player inventory into storage counts.
Change Scope
- Exclude player inventory contributions when player is currently in RTS craft terminal menu.
Acceptance
- In RTS craft terminal linked panel, only linked-storage entries are listed.
- Outside terminal (RTS overlay), existing behavior remains unchanged unless specified.

Item 7 - Drag/drop import into storage grid (crafting + RTS terminal)
Symptom
- Returning carried item to linked storage needs button flow; cannot directly drag to storage grid.
Change Scope
- Allow mouse release of carried stack over storage grid area to trigger import.
- Apply to both RTS terminal screen and RTS overlay-on-container crafting flow.
Acceptance
- Pick up stack, drag to storage grid, release -> imports to linked storage.

Item 8 - Chain-build target logic should lock air cells, not edge-based fallback
Symptom
- Chain build can shift upward when environment changes because placement resolves to adjacent face.
Change Scope
- Shape placement should target intended replaceable air cells.
- If target cell becomes occupied, skip that cell (do not place on top/adjacent fallback).
Acceptance
- During shape confirm, occupied targets are skipped, not redirected to top/adjacent cells.

Item 9 - PageUp/PageDown should control shape dimensions (L/W/H), including negative extension
Symptom
- Currently PageUp/PageDown only nudges cube height stage.
Change Scope
- Add keyboard nudges for footprint dimensions as well.
- Allow negative direction extension.
- Keep mouse shaping usable simultaneously with keyboard nudges.
Acceptance
- After setting start point, PgUp/PgDn can nudge dimensions.
- Shift/Ctrl modifier mapping documented in code comments/UI text.
- Negative dimension values extend opposite direction.

Item 10 - JEI auto-fill should partial-fill even when materials are missing
Symptom
- If one ingredient is missing, current transfer aborts full fill.
Change Scope
- JEI transfer should fill available slots/items without all-or-nothing rollback.
Acceptance
- Missing ingredients leave holes, but available ingredients are still inserted.

Item 11 - Search mode should not interfere with storage block selection/usage
Symptom
- Entering search mode interferes with selecting/locking usage flow.
Change Scope
- Search focus should not block core selection/place interactions unnecessarily.
Acceptance
- User can continue selecting/using storage items without needing explicit search exit.

Execution order
1) Core interaction and hotbar/funnel fixes
2) JEI behavior fixes
3) Terminal display/drag-import improvements
4) Shape targeting + dimension control
5) Compile check and regression pass
